# Synthetic Methodology Routing Case

Synthetic Example — For Portfolio Demonstration Only.

## Input

"A new portfolio project needs to explain an AI-assisted project memory system for graduate admissions. It should show workflow design, responsible knowledge governance, and enterprise relevance without exposing private notes."

## Problem Type

Mixed portfolio and system-design problem with three possible categories:

- AI product design
- Knowledge governance
- Enterprise workflow

## Suggested Methodology

Primary route: knowledge governance

Secondary route: enterprise workflow

Supporting route: AI product design

## Why This Route

The input is primarily about presenting a knowledge system safely. The highest-risk requirement is not interface design or feature scope, but deciding what can be made public, what must remain private, and how examples should be structured without leaking sensitive information.

Enterprise workflow is a secondary route because the project also needs to explain how structured knowledge, decision logs, execution reports, and review loops could translate into organizational knowledge management.

AI product design is a supporting route because the future cockpit/interface should make the workflow usable, but the immediate task is documentation and governance.

## Expected Output

- A public-safe repository structure
- Clear public/private boundary rules
- Synthetic decision and execution artifacts
- A methodology routing explanation
- Reviewable documentation for admissions readers

## Human Override Check

The human reviewer should confirm:

- No private notes or project records are included.
- No client, financial, trading-sensitive, credential, or personal journal data appears.
- The example does not imply that the system is a fully deployed enterprise product.
- The final wording remains accurate for a portfolio case.

## Final Routing Decision

Route the project through knowledge governance first, then enterprise workflow, with AI product design used only for future cockpit and interface planning.

## Write-back Result

The routing decision should be written back to the methodology library as a reusable pattern: when a portfolio project involves AI-assisted knowledge work and sensitive boundaries, apply knowledge governance before product-design presentation.
