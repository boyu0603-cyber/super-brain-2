# Methodology Router

## What It Is

The methodology router is a conceptual layer that helps match a new input, project issue, or decision question with a suitable workflow or thinking method.

It answers a practical question: "What kind of problem is this, and which method should guide the next step?"

## Why It Matters

AI-assisted work can become inconsistent if every problem is handled with a generic prompt or ad hoc conversation. A methodology router creates a more disciplined process.

It helps:

- Reduce scattered reasoning
- Reuse proven project methods
- Improve decision consistency
- Make work easier to audit
- Separate different categories of thinking

## Example Routing Categories

- AI product design
- Enterprise workflow
- Trading research
- Content production
- Knowledge management
- Project review
- Risk and incident review

## Human Override

The methodology router provides a recommendation, not a command.

The human operator can override the suggested route when:

- The input has hidden context
- The risk level is higher than expected
- The category is ambiguous
- The method is outdated
- A sensitive boundary is involved

## Public-Safe Boundary

This public version does not include private prompts, sensitive routing rules, commercial project details, or trading-sensitive materials.

All routing examples are synthetic and designed only to explain the concept.
