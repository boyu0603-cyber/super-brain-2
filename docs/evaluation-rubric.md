# Evaluation Rubric

This rubric evaluates Super Brain 2.0 as a knowledge and decision-management system.

| Criterion | Description | Strong Evidence |
| --- | --- | --- |
| Retrievability | Important knowledge can be found when needed. | Clear structure, tags, project links, search readiness |
| Decision clarity | Decisions are explicit rather than hidden in notes. | Decision logs include rationale, date, owner, and status |
| Source traceability | Important claims and inputs preserve their origin. | Entries include source type, date, and review state |
| Methodology reuse | Useful methods can be applied across projects. | Methodology library, routing table, reusable templates |
| Privacy boundary | Public and private information are separated. | Clear exclusion rules and public-safe review process |
| Workflow consistency | Similar work follows similar steps. | Capture, structure, route, decide, execute, write-back loop |
| Human review readiness | AI outputs are reviewed before decisions or publication. | Human approval stage and review checklist |
| Evidence of use | The system includes synthetic but realistic examples showing how decisions, execution reports, and methodology routing work. | Sample decision records, sample execution reports, and routing cases are included without exposing private data. |
| Enterprise adaptability | The system can be adapted beyond personal use. | Team workflow concepts, governance model, auditability |

## Scoring Approach

Each criterion can be scored from 1 to 5:

- 1: Weak or missing
- 2: Present but inconsistent
- 3: Functional baseline
- 4: Strong and repeatable
- 5: Mature and adaptable

## Example Use

The rubric can be used during a monthly review to identify which part of the system needs improvement. For example, strong capture but weak source traceability would indicate that raw information is being stored, but not yet governed well enough for reliable reuse.

## Portfolio Interpretation

For this public repository, the rubric demonstrates the design logic of the system. It does not claim that every criterion has reached enterprise maturity.
